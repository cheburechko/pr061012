enum ActionType

The ActionType enum. In some cases there are ids in parameters. These ids denote index of object in participants array. Example: action MINE, actor is Humanoid, participants are Resource (index is 0) and Tool (PICKAXE) (index is 1), parameters are res_id (= 0) and tool_id (= 1).

Enumerator:
GO 

Description: Object goes to to another point (direction is set by angle).
Actor: Creature/Weather
Participants: none
Parameters: angle (double), speed (Speed)

MOVE 

Description: Object moves another object.
Actor: Humanoid
Participants: Object
Parameters: angle (double), speed (Speed)

MINE_OBJ 

Description: Resource mining by tool.
Actor: Humanoid
Participants: Resource, Tool (PICKAXE)
Parameters: res_id (uint), tool_id (uint)

REPAIR_OBJ 

Description: Repair building.
Actor: Humanoid
Participants: Building, Tool
Parameters: building_id (uint), tool_id (uint)

HARM_OBJS 

Description: Harm one or more objects.
Actor: Creature/Weather
Participants: several objects
Parameters: none.

CREATE_OBJ 

Description: Create object.
Actor: Humanoid/Resource
Participants: none
Parameters:

  • obj_type (ObjectType), x (uint) and y (uint)
  • for Resource creation: res_type (ResourceType), res_amount (uint)
  • for Tool creation: tool_type (ToolType), mat_type (uint)
  • for Creature creation: creat_type (CreatureType)
DROP_OBJS 

Description: Drop objects to ground.
Actor: Creature
Participants: several objects
Parameters: none.

PICK_UP_OBJS 

Description: Pick up objects from the ground.
Actor: Creature
Participants: several objects
Parameters: none.

EAT_OBJ 

Description: Object eats object.
Actor: NonHumanoid
Participants: Resource
Parameters: none

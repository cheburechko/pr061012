enum ActionType

The ActionType enum. In some cases there are ids in parameters. These index denote index of object in participants array. Example: action MINE, actor is Humanoid, participants are Resource (index is 0) and Tool (PICKAXE) (index is 1), parameters are res_index (= 0) and tool_index (= 1).

NOTE: Don't forget to increase AMNT_ACTION_TYPES while adding new action types!

Enumerator:
GO 

Description: Object goes to to another point (direction is set by angle).
Actor: Creature/Weather
Participants: none
Parameters: angle (double), speed (SpeedType)

MOVE 

Description: Object moves another object.
Actor: Humanoid
Participants: Object
Parameters: angle (double), speed (SpeedType)

MINE_OBJ 

Description: Resource mining by tool.
Actor: Humanoid
Participants: Resource, Tool (PICKAXE)
Parameters: res_index (uint), tool_index (uint)

REGENERATE_OBJ 

Description: Regenerate object.
Actor: Humanoid/Resource
Participants: Object, Tool
Parameters: object_index (uint), tool_index (uint)

HARM_OBJS 

Description: Harm one or more objects.
Actor: Creature/Weather
Participants: Tool (Creature only, optional), several objects
Parameters: tool_index (uint)

CREATE_OBJ 

Description: Create object.
Actor: Humanoid/Resource
Participants: none
Parameters:

  • obj_type (ObjectType)
  • for Resource creation: res_type (ResourceType), res_amount (uint)
  • for Tool creation: tool_type (ToolType), mat_type (uint)
  • for Creature creation: creat_type (CreatureType)
  • for Building creation: building_max_space (uint), building_max_health (uint)
DROP_OBJS 

Description: Drop objects to ground.
Actor: Creature
Participants: several objects
Parameters: none.

PICK_UP_OBJS 

Description: Pick up objects from the ground.
Actor: Creature
Participants: several objects
Parameters: none.

EAT_OBJ 

Description: Object eats object.
Actor: NonHumanoid
Participants: Resource
Parameters: none
.
